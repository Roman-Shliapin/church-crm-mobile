## Авторизація

Захищені ендпоінти потребують заголовок:
Authorization: Bearer <token>
Токен отримується при логіні або реєстрації. Термін дії — **7 днів**.

🔒 — захищений токеном  
🔐 — захищений токеном + роль `admin`

## Auth

### Реєстрація

POST /auth/register
**Логіка:** шукає користувача по `phone` в колекціях `members` і `candidates`. Якщо знайдено і пароля ще немає — додає `email` і хешований `password`.

**Body:**

```json
{
  "phone": "+380507551884",
  "email": "user@gmail.com",
  "password": "123456"
}
```

**Відповідь 201:**

```json
{
  "token": "eyJ...",
  "name": "Кальчук Євгенія Юріївна",
  "role": "user"
}
```

**Помилки:**
| Код | Повідомлення |
|-----|-------------|
| 404 | Користувача з таким номером не знайдено |
| 400 | Акаунт вже зареєстровано |
| 500 | Помилка сервера |

---

POST /auth/login
**Body:**

```json
{
  "email": "user@gmail.com",
```

**Відповідь 200:**

```json
{
  "token": "eyJ...",
  "name": "Кальчук Євгенія Юріївна",

```

**Помилки:**
| Код | Повідомлення |

```json
    "date": "27.04.2026, 13:31:22",

    "archived": false,
    "createdAt": "2026-04-27T11:31:22.943Z",

```

---

### Одна заявка за id

GET /needs/:id  
🔒 Потребує токен  
**Параметр:** `:id` — MongoDB `_id` заявки (рядок).

**Відповідь 200:** об’єкт заявки (як у елементі масиву з `GET /needs`, плюс за наявності поля `repliedAt`, `replyMessage`, `doneAt`, `doneMessage`).

**Помилки:**
| Код | Повідомлення (приклад) |
|-----|------------------------|
| 401 | (скидання сесії в додатку) |
| 403 | Немає доступу до цієї заявки |
| 404 | Заявку не знайдено |
| 500 | Помилка сервера |

**Важливо для бекенду:** у моделі `Need` поле `userId` зазвичай збігається з **числовим telegram `id`** з документа `Member` / `Candidate`, а не з Mongo `_id`. Після `const user = await getUser(req.user.id)` перевіряйте власника так: **`need.userId === user.id`** (telegram id), а не порівняння з `user._id`, інакше сервер помилково відповість «Користувача не знайдено» або відхилить доступ.

### Створити заявку

POST /needs
🔒 Потребує токен

**Body:**

```json
{
  "description": "Потребую гуманітарної допомоги",
  "type": "humanitarian"
}
```

**Поле `type`:**

- `"humanitarian"` — гуманітарна допомога (продукти / хімія)
- `"other"` — інше

**Відповідь 201:**

```json
{
  "_id": "69ef490a20ec2c51468181e9",
  "userId": 987730182,
  "name": "Кальчук Євгенія Юріївна",
  "baptism": "24-05-2015",
  "birthday": "19-02-1987",
  "phone": "+380507551884",
  "description": "Потребую гуманітарної допомоги",
  "type": "humanitarian",
  "date": "27.04.2026, 13:31:22",
  "status": "нова",
  "archived": false,
  "createdAt": "2026-04-27T11:31:22.943Z"
}
```

**Помилки:**
| Код | Повідомлення |
|-----|-------------|
| 400 | Опис є обовʼязковим |
| 401 | Не авторизовано |
| 404 | Користувача не знайдено |
| 500 | Помилка сервера |

🔒 Потребує токен

**Params:** `id` — MongoDB `_id` заявки

**Логіка:** повертає заявку тільки якщо вона належить авторизованому користувачу.

**Відповідь 200:**

```json

  "userId": 987730182,
  "name": "Кальчук Євгенія Юріївна",
  "description": "Потребую гуманітарної допомоги",
  "type": "humanitarian",
  "date": "27.04.2026, 13:31:22",
  "status": "нова",
  "archived": false,
  "createdAt": "2026-04-27T11:31:22.943Z",
  "repliedAt": "2026-01-19T14:45:27.306Z",
```

## Адмін — Заявки

### Всі активні заявки

GET /admin/needs/active
🔐 Потребує токен + роль `admin`

**Відповідь 200:** масив заявок де `archived: false`, відсортованих від нових до старих.

---

### Всі архівні заявки

GET /admin/needs/archived
🔐 Потребує токен + роль `admin`

**Відповідь 200:** масив заявок де `archived: true`, відсортованих від нових до старих.

---

### Змінити статус заявки

PATCH /admin/needs/:id/status
🔐 Потребує токен + роль `admin`

**Params:** `id` — MongoDB `_id` заявки

**Body:**

```json
{
  "action": "in_progress"
}
```

**Можливі значення `action`:**
| Значення | Новий статус | Додаткові дії |
|----------|-------------|---------------|
| `in_progress` | `"в очікуванні"` | Записує `inProgressAt`, `waitingAt` |
| `done` | `"виконано"` | Архівує (`archived: true`), записує `doneAt` |

**Відповідь 200:** оновлений обʼєкт заявки

**Помилки:**
| Код | Повідомлення |
|-----|-------------|
| 400 | Невідома дія |
| 404 | Заявку не знайдено |
| 500 | Помилка сервера |

---

### Відповісти на заявку

POST /admin/needs/:id/reply
🔐 Потребує токен + роль `admin`

**Params:** `id` — MongoDB `_id` заявки

**Body:**



**Помилки:**

|-----|-------------|
| 400 | Повідомлення є обовʼязковим |
| 404 | Заявку не знайдено |
| 500 | Помилка сервера |
